搭建博客的步骤

以“Hexo + GitHub Pages”为例

这个方案写文章就像写代码一样，用最纯粹的Markdown格式，非常有极客范儿。整个过程可以分为四步：

一、安装基础软件

1、Node.js (Hexo的运行环境)。Node.js安装教程_哔哩哔哩_bilibili

2、Git (版本控制和上传工具)。在Windows上安装Git_哔哩哔哩_bilibili

3、以及一个趁手的代码编辑器（如VS Code）。

二、本地生成

1、打开命令行，输入npm install -g hexo-cli装上Hexo。

2、命令行输入hexo init myblog创建一个叫“myblog”的博客文件夹。

3、输入hexo server就能在浏览器里通过http://localhost:4000看到你新鲜出炉的博客了 ！

三、部署上线

1、去 GitHub 注册个账号，创建一个名为 你的用户名.github.io 的仓库。记得要写readme。

2、在你本地的博客文件夹里（C:\Users\ricky\myblog），简单配置一下_config.yml文件，填上仓库地址。

  • 你的用户名 要替换成你 GitHub 的真实用户名
  • 比如你 GitHub 用户名是 ricky123，那么 repo 就是：https://github.com/ricky123/ricky123.github.io.git
  • 冒号后面要有一个空格（YAML 格式要求，很重要！）
  • branch 一般写 master，如果你的 GitHub 默认分支是 main，就改成 main

3、ctrl+c关闭本地服务：hexo server 启动后，它会一直占用着命令行窗口，你无法输入新命令。更重要的是，它会把生成的网站文件锁住，这时候如果运行部署命令，会导致文件冲突或部署失败。

4、安装Git部署插件：npm install hexo-deployer-git --save

5、最后运行hexo clean && hexo generate && hexo deploy，你的博客就被自动打包上传到这个“家”里了。稍等片刻，访问你的用户名.github.io，就能看到你的博客成功上线！

  • 注意：部署前为了防止网络错误，配置git代理（端口为7890）

四、内容创作与美化

  • 写文章：在source/_posts文件夹里新建Markdown文件（比如my-first-post.md），用你最熟悉的Markdown语法写下第一篇学习心得。文章开头可以用---包裹起来的内容来定义标题、日期、分类等信息 。
  • 个人简介：你可以新建一个“关于”页面，或者在首页专门开辟一个板块来介绍自己。关于页可以包含信息卡片、个人简介、技术栈、工作经历和教育经历等模块，让访客全面了解你 。
  • 换主题：Hexo有成千上万款精美主题。比如，你可以去GitHub上把流行的 NexT 主题下载到themes文件夹，然后在_config.yml里把主题名改成next，博客瞬间就能变个样 。

写博客的流程

步骤 命令 说明
1 hexo new "文章标题" 创建新文章
2 编辑 source/_posts/文章标题.md 用 Markdown 写内容
3 hexo server 本地预览
4 Ctrl+C 关闭本地服务
5 hexo clean && hexo generate && hexo deploy 发布上线
  • 步骤1，确保在…\myblog路径下执行
  • 步骤3，对应地址http://localhost:4000

插图方式

一、全局资源文件夹

（适合图片数量不多，且多篇文章共用）

  1. 创建文件夹：在你的博客 source 文件夹下，手动创建一个 images 文件夹（路径：C:\Users\ricky\myblog\source\images）。

  2. 放入图片：把你文章里要用的所有图片都复制到这个 images 文件夹里。

  3. 引用图片：在 Markdown 文章里，用 相对根目录的路径 来引用

注意：图片格式可以通过文件夹-查看-显示后缀进行修改

二、文章资源文件夹

（推荐，图片与文章一一对应，管理更清晰）

0、安装图片插件：

1、开启配置：打开博客根目录的 _config.yml 文件，找到 post_asset_folder 这个配置项，把它从 false 改成 true

2、创建新文章：之后，当你使用 hexo new "文章标题" 创建新文章时，Hexo 不仅会在 source/_posts 下生成一个 .md 文件，还会同时生成一个与文章同名的文件夹。例如，你创建“我的第二篇博客”，就会生成：

  • source/_posts/我的第二篇博客.md
  • source/_posts/我的第二篇博客/ （这是一个空文件夹）

3、存放图片：把你这篇文章需要用到的所有图片，都放进这个同名的文件夹里。

4、Typora 设置与引用：

  • 在 Typora 中打开你的 .md 文章。
  • 将图片直接拖拽或粘贴到文章中。这时你需要根据你的设置来确保路径正确。
  • Typora 中， 格式 → 图像 → 设置图片根目录，然后选择你的博客 我的第二篇博客 文件夹。
  • Typora 中， 格式 → 图像 → 全局图像设置设置并勾选：插入图片时复制到指定路径；优先使用相对路径；为相对路径添加./
  • 设置好根目录后，你在文章中插入图片时，Typora 生成的路径就会是 ![图片描述](文章同名文件夹/图片名.jpg)，例如 ![abc.png](./abc.png)。这个路径无论是在本地 Typora 预览，还是经过 Hexo 生成后部署到网上，都能被正确解析。
