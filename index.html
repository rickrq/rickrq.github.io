搭建博客的过程

详细步骤：以“Hexo + GitHub Pages”为例

这个方案写文章就像写代码一样，用最纯粹的Markdown格式，非常有极客范儿。整个过程可以分为四步：

  1. 准备工作：搭个“作坊”
    你需要在电脑上安装三个基础软件：Node.js (Hexo的运行环境)、Git (版本控制和上传工具)，以及一个趁手的代码编辑器（如VS Code）。安装好这些，建站的基础就有了 。

  2. 本地生成：种下博客的“种子”
    打开命令行，输入npm install -g hexo-cli装上Hexo，然后用hexo init myblog创建一个叫“myblog”的博客文件夹，再用hexo server就能在浏览器里通过http://localhost:4000看到你新鲜出炉的博客了 ！

  3. 部署上线：把“家”搬到网上
    想让全世界看到，就需要一个“家”。去 GitHub 注册个账号，创建一个名为 你的用户名.github.io 的仓库。接着，在你本地的博客文件夹里，简单配置一下_config.yml文件，填上仓库地址。最后运行hexo clean && hexo generate && hexo deploy，你的博客就被自动打包上传到这个“家”里了。稍等片刻，访问你的用户名.github.io，就能看到你的博客成功上线！

    • 注意：部署前为了防止网络错误，配置git代理（端口为7890）

  4. 内容创作与美化：用心经营你的“小家”

    • 写文章：在source/_posts文件夹里新建Markdown文件（比如my-first-post.md），用你最熟悉的Markdown语法写下第一篇学习心得。文章开头可以用---包裹起来的内容来定义标题、日期、分类等信息 。
    • 个人简介：你可以新建一个“关于”页面，或者在首页专门开辟一个板块来介绍自己。关于页可以包含信息卡片、个人简介、技术栈、工作经历和教育经历等模块，让访客全面了解你 。
    • 换主题：Hexo有成千上万款精美主题。比如，你可以去GitHub上把流行的 NexT 主题下载到themes文件夹，然后在_config.yml里把主题名改成next，博客瞬间就能变个样 。
